# 数据运营与展示产品化计划

## 背景

当前项目已经完成数据平台基础架构、多数据源接入、管理端认证、采集任务可靠性、小程序 API 契约和小程序基础页面。若下一阶段暂不优先处理真实小程序验收和真实数据采集问题，功能开发应从“链路可用”转向“数据可运营、展示更像产品、配置可持续演进”。

本阶段围绕四个方向展开：

1. 管理端数据运营工作台。
2. 小程序展示体验增强。
3. 数据模型和查询能力优化。
4. 管理配置化能力。

## 阶段目标

- 让管理端从任务列表和数据源列表升级为数据运营人员可持续使用的工作台。
- 让小程序端从基础展示升级为支持对比、解释、收藏和更清晰指标组织的展示产品。
- 为更多展示玩法补齐指标、区域、时间维度和聚合查询能力。
- 将指标展示、首页推荐、质量阈值和调度默认值逐步配置化，减少后续硬编码修改。

## 当前实施状态

已完成首批纵向闭环：

- 管理端新增数据源详情、任务详情、质量报告详情、审核数据源筛选、指标展示配置和运营配置编辑入口。
- 后端新增指标元数据、区域层级字段、应用配置表、详情 API、指标配置 API、质量规则配置读取和小程序聚合 API。
- 小程序新增指标分组、首页推荐、收藏城市指标组合、城市详情聚合展示和趋势页城市对比。
- 测试覆盖管理端详情 API、指标配置 API、配置 API、质量规则配置化、小程序指标分组、城市详情聚合、对比趋势和页面 smoke 路径。

本阶段仍未覆盖真实小程序开发者工具验收和真实采集跑批，这两项按前置约定暂不纳入本计划实施范围。

## 工作项一：管理端数据运营工作台

### 目标

把管理端从“能管理数据源和任务”升级为“能完成数据运营、审核、追踪和问题定位”的工作台。

### 实施内容

- 新增数据源详情页或详情抽屉：
  - 基础信息。
  - 入口 URL。
  - 数据源类型。
  - 最近采集任务。
  - 最近质量报告。
  - 调度策略。
  - 手动触发、重试、停用入口。
- 新增任务详情页或详情抽屉：
  - 当前状态。
  - 执行耗时。
  - 锁信息。
  - 重试次数。
  - 错误类型和错误摘要。
  - 导入统计。
- 增强数据审核工作台：
  - 按指标、区域、周期、状态、数据源筛选。
  - 支持批量发布、驳回、修改状态。
  - 支持查看单条数据变更历史。
  - 支持编辑值时填写复核原因。
- 增强质量报告详情：
  - 按 error 和 warning 分组展示。
  - 明确指标、区域、周期、维度和值。
  - 给出处理建议，例如复核、驳回、暂缓发布。

### 建议交付物

- 管理端数据源详情视图。
- 管理端任务详情视图。
- 审核工作台筛选和批量操作优化。
- 质量报告详情视图。
- 后端详情 API 和筛选参数补强。

### 验收标准

- 运营人员可以从数据源入口定位最近任务、质量问题和可执行操作。
- 失败任务可以在详情中看到错误摘要、重试次数和导入统计。
- 审核工作台可以完成筛选、批量发布、驳回和复核原因记录。
- 质量报告明细能定位到具体指标、区域、周期和维度。
- 管理端构建通过，关键数据操作有测试或明确手工验证记录。

## 工作项二：小程序展示体验增强

### 目标

把小程序从“能查看数据”升级为“用户能理解数据、比较数据、保存常用视角”的展示产品。

### 实施内容

- 指标分组：
  - 房价指标。
  - CPI 指标。
  - 后续经济指标预留分组。
- 指标解释：
  - 展示名称。
  - 单位。
  - 口径说明。
  - 更新时间。
  - 数据来源。
- 城市对比：
  - 支持选择 2 到 3 个城市。
  - 支持同一指标趋势对比。
  - 支持清空和切换对比城市。
- 收藏增强：
  - 收藏城市。
  - 收藏指标。
  - 收藏城市与指标组合。
  - 首页优先展示收藏组合。
- 趋势图组件化：
  - 统一趋势图输入结构。
  - 支持单城市趋势。
  - 支持多城市对比。
  - 预留不同指标单位和精度格式化。

### 建议交付物

- 小程序指标分组展示。
- 指标说明组件。
- 城市对比页面或趋势页对比模式。
- 收藏城市和指标组合的数据结构。
- 统一趋势图组件或图表封装。

### 验收标准

- 用户可以按指标分类浏览房价和 CPI 数据。
- 指标页面或卡片能解释指标含义、单位和更新时间。
- 趋势页支持至少两个城市同指标对比。
- 收藏城市和指标组合后，首页能优先展示。
- 图表组件可以被城市详情、趋势页和对比视图复用。

## 工作项三：数据模型和查询能力优化

### 目标

增强数据模型表达力和 API 查询能力，支撑更丰富的小程序展示与管理端筛选分析。

### 实施内容

- 增加指标元数据：
  - 分类。
  - 展示名称。
  - 单位。
  - 描述。
  - 展示精度。
  - 排序。
  - 默认维度。
  - 是否在小程序展示。
- 增强区域层级：
  - 全国。
  - 省份。
  - 城市。
  - 父子区域关系。
  - 区域排序和展示开关。
- 增强时间维度查询：
  - 最新一期。
  - 近 6 期。
  - 近 12 期。
  - 同比变化。
  - 环比变化。
- 增加展示聚合接口：
  - 指标分组接口。
  - 城市详情聚合接口。
  - 城市对比趋势接口。
  - 首页推荐内容接口。
- 优化查询性能：
  - 为常用筛选字段补充索引。
  - 控制小程序接口返回字段。
  - 减少小程序多次请求拼装数据。

### 建议交付物

- 指标元数据 migration 和 repository 方法。
- 区域层级字段或关系表。
- 小程序聚合 API。
- 管理端筛选 API 增强。
- 查询能力测试。

### 验收标准

- 指标具备分类、排序、精度和展示开关。
- 区域可以表达全国、省、市层级。
- 小程序可以通过聚合接口获取城市详情和对比趋势所需数据。
- 管理端可以按指标分类、区域层级和周期筛选。
- 测试覆盖指标元数据、区域层级和聚合查询路径。

## 工作项四：管理配置化能力

### 目标

把展示、质量和默认策略中频繁变化的内容从代码硬编码迁移到可管理配置，降低后续新增指标和调整展示的成本。

### 实施内容

- 指标展示配置：
  - 是否展示。
  - 展示名称。
  - 单位。
  - 排序。
  - 精度。
  - 默认图表类型。
- 首页内容配置：
  - 推荐指标。
  - 推荐城市。
  - 热门排行指标。
  - 默认趋势指标。
- 数据源默认配置：
  - 默认调度间隔。
  - 默认重试次数。
  - 默认超时时间。
  - 默认质量规则集。
- 质量规则配置：
  - 数值范围阈值。
  - 缺失数据规则。
  - warning 和 error 分级。
  - 不同数据源类型的规则覆盖。
- 管理端配置页：
  - 指标配置。
  - 首页配置。
  - 质量规则配置。
  - 数据源默认策略配置。

### 建议交付物

- 配置表或结构化配置模型。
- 配置读取 repository/service。
- 管理端配置页面。
- 配置变更校验。
- 配置默认值和回退策略文档。

### 验收标准

- 常用展示字段不需要改代码即可调整。
- 首页推荐指标和推荐城市可以在管理端配置。
- 质量规则阈值可以按数据源类型调整。
- 配置缺失时系统能使用默认值继续运行。
- 配置变更有基础校验，避免写入无效展示状态。

## 推荐里程碑

### 里程碑一：管理端运营工作台

优先补齐运营人员最需要的详情、审核和问题定位能力。

交付物：

- 数据源详情。
- 任务详情。
- 审核工作台增强。
- 质量报告详情。

### 里程碑二：指标元数据与展示聚合

在展示体验升级前，先补齐指标、区域和聚合查询能力。

交付物：

- 指标元数据。
- 区域层级。
- 城市详情聚合 API。
- 城市对比趋势 API。

### 里程碑三：小程序体验增强

基于稳定聚合接口优化小程序真实产品体验。

交付物：

- 指标分组。
- 指标解释。
- 城市对比。
- 收藏组合。
- 趋势图组件化。

### 里程碑四：配置化管理

最后将展示和规则中的高频变化项迁移到管理配置。

交付物：

- 指标展示配置。
- 首页内容配置。
- 质量规则配置。
- 数据源默认策略配置。

## 推荐执行顺序

1. 管理端数据运营工作台。
2. 数据模型和查询能力优化。
3. 小程序展示体验增强。
4. 管理配置化能力。

优先级理由：先让已有数据具备可运营能力，再补齐展示所需的模型和查询基础；随后升级小程序体验，最后把展示和规则配置化，避免过早做配置系统导致抽象不稳定。
