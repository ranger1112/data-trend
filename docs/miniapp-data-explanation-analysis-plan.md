# 小程序数据解释与分析能力增强计划

## 背景

当前项目已经具备数据抓取、管理和小程序展示的基础闭环；管理端也已经补齐操作日志、批量预览、配置历史、diff 预览和回滚能力。下一阶段如果仍暂不考虑真实小程序验收和真实数据采集跑批，功能开发应优先提升小程序端的数据解释力和分析价值，同时继续降低管理端配置误改风险。

本计划围绕四个方向展开：

1. 小程序指标解释增强。
2. 趋势分析与变化摘要。
3. 城市对比体验优化。
4. 管理端配置表单化。

## 阶段目标

- 让用户不仅能看到数据，还能理解指标含义、统计口径和适用场景。
- 让趋势页从“展示折线”升级为“解释变化”。
- 让城市对比从“并列数据”升级为“差异结论”。
- 让运营配置从 JSON textarea 逐步升级为可控表单，并复用现有 preview/history/rollback 能力。

## 工作项一：小程序指标解释增强

### 目标

为小程序展示的核心指标补充解释信息，让用户理解每个指标是什么、怎么看、适合用于什么场景。

### 实施内容

- 扩展指标元数据：
  - 指标含义。
  - 统计口径。
  - 单位说明。
  - 更新频率。
  - 适用场景。
  - 注意事项。
- 后端 mini API 输出指标解释字段。
- 小程序首页指标卡展示简短说明。
- 小程序趋势页展示完整指标说明。
- 小程序城市详情页展示指标口径提示。
- 管理端指标配置中支持维护解释字段。

### 建议交付物

- 指标解释字段 model/migration。
- 指标序列化与 mini API 响应扩展。
- 管理端指标解释编辑入口。
- 小程序指标说明组件。
- API 与小程序测试。

### 验收标准

- 小程序首页可以看到核心指标的短说明。
- 趋势页可以查看指标含义、单位和口径。
- 城市详情页的指标卡能展示必要解释。
- 管理端可以维护指标解释字段。
- 指标解释字段有测试覆盖，缺省值不会导致页面异常。

## 工作项二：趋势分析与变化摘要

### 目标

在已有趋势数据基础上增加派生分析，让用户快速理解趋势方向、变化幅度和近期波动。

### 实施内容

- 后端趋势 API 增加分析摘要：
  - 最新值。
  - 上期值。
  - 变化值。
  - 变化百分比。
  - 趋势方向：上涨、下降、持平。
  - 近 N 期最高值。
  - 近 N 期最低值。
  - 近 N 期均值。
  - 波动提示。
- 支持查询窗口参数，例如 `window=6`、`window=12`。
- 小程序趋势页展示变化摘要。
- 小程序趋势页展示近期高低点和趋势文案。
- 对数据不足的场景返回明确空态。

### 建议交付物

- 趋势分析 helper/service。
- mini 趋势 API 响应 schema 扩展。
- 小程序趋势摘要 UI。
- 趋势分析测试。
- 数据不足和异常值场景测试。

### 验收标准

- 趋势 API 可以返回最新值、上期值和变化方向。
- 小程序趋势页能展示“上涨/下降/持平”摘要。
- 近 N 期最高、最低和均值计算正确。
- 数据不足时页面展示稳定空态。
- 房价指标和 CPI 指标都能复用同一套分析逻辑。

## 工作项三：城市对比体验优化

### 目标

让城市对比不只展示多条曲线，还能给出城市之间的差值、排名和结构化结论。

### 实施内容

- 后端城市对比 API 增加对比摘要：
  - 城市最新值。
  - 城市间差值。
  - 城市间差值百分比。
  - 当前排名。
  - 领先/落后城市。
  - 变化方向对比。
- 支持对比结果按指标、住宅类型和面积类型过滤。
- 小程序对比页展示城市差异摘要。
- 小程序对比页展示领先/落后提示。
- 小程序对比页强化空态和加载态。

### 建议交付物

- 城市对比分析 service。
- mini 对比 API 响应 schema 扩展。
- 小程序对比摘要组件。
- 小程序对比页交互优化。
- 城市对比 API 与小程序测试。

### 验收标准

- 对比 API 可以返回城市差值和排名。
- 小程序对比页可以展示领先/落后摘要。
- 多城市、多指标维度下结果稳定。
- 缺少某城市数据时有明确提示。
- 对比页测试覆盖核心状态切换和空态。

## 工作项四：管理端配置表单化

### 目标

降低运营人员直接编辑 JSON 的风险，把核心配置逐步升级为结构化表单，同时继续复用已完成的 diff 预览、历史版本和回滚能力。

### 实施内容

- `miniapp.home` 表单化：
  - 推荐指标选择。
  - 推荐城市选择。
  - 默认排行指标。
  - 默认趋势指标。
- `quality.rules` 表单化：
  - 数据源类型规则选择。
  - 期望城市数。
  - 期望指标数。
  - 必填维度。
  - 值域阈值。
  - warning 阈值。
- `data_source.defaults` 表单化：
  - 默认重试次数。
  - 默认超时时间。
  - 默认调度间隔。
- 表单提交前调用现有配置 preview API。
- 表单保存后继续生成配置历史。
- 表单回滚继续复用现有历史面板。

### 建议交付物

- 管理端配置表单组件。
- 配置 key 到表单 schema 的映射。
- 表单值与 JSON 配置转换逻辑。
- 配置预览确认交互复用。
- 前端构建验证和核心交互测试。

### 验收标准

- `miniapp.home` 可以通过表单编辑并保存。
- `quality.rules` 可以通过表单编辑主要规则字段。
- `data_source.defaults` 可以通过表单编辑默认策略。
- 保存前能看到 diff 预览。
- 保存后能在配置历史中看到版本。
- 回滚后表单能正确回显历史版本值。

## 推荐里程碑

### 里程碑一：指标解释打底

先补指标解释字段、API 输出和小程序展示，让用户能理解基础数据。

交付物：

- 指标解释字段。
- mini API 响应扩展。
- 小程序指标说明展示。
- 指标解释测试。

### 里程碑二：趋势分析摘要

在趋势 API 和小程序趋势页增加变化分析。

交付物：

- 趋势分析 helper。
- 趋势摘要 API 字段。
- 小程序趋势摘要 UI。
- 趋势分析测试。

### 里程碑三：城市对比结论

增强城市对比 API 和页面，让对比结果更容易读。

交付物：

- 城市对比摘要。
- 排名和差值字段。
- 小程序对比摘要展示。
- 对比测试。

### 里程碑四：配置表单化

把高频配置从 JSON 编辑升级为表单编辑。

交付物：

- 首页推荐配置表单。
- 质量规则配置表单。
- 数据源默认策略表单。
- preview/history/rollback 复用验证。

## 推荐执行顺序

1. 小程序指标解释增强。
2. 趋势分析与变化摘要。
3. 城市对比体验优化。
4. 管理端配置表单化。

优先级理由：先提升小程序端用户理解数据的能力，再补趋势和对比分析，最后利用已有配置历史与回滚基础优化管理端配置体验。这样可以最快把当前“可展示”的产品推进到“可解释、可分析、可运营”。
