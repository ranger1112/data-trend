# 产品可用性深化计划

## 背景

当前项目已经完成数据运营与展示产品化首轮闭环：管理端具备数据源、任务、质量报告、指标和配置管理能力；小程序具备指标分组、首页推荐、收藏组合、城市详情和城市对比能力；后端已经补齐指标元数据、区域层级、配置表和聚合 API。

若下一阶段仍暂不考虑真实小程序验收和真实数据采集跑批，功能开发应继续聚焦“运营效率、数据解释力、权限协作、前端体验细化和质量保障”。

本阶段围绕五个方向展开：

1. 管理端协作与审计增强。
2. 小程序数据解释和用户体验增强。
3. 数据查询与分析能力增强。
4. 配置中心产品化。
5. 测试与质量保障增强。

## 阶段目标

- 让管理端操作可追踪、可回看、可审计，支撑多人协作。
- 让小程序用户更容易理解指标、趋势和城市对比结果。
- 让后端提供更强的派生分析和时间窗口查询能力。
- 将当前 JSON 配置编辑升级为更可控的表单化配置中心。
- 为管理端、小程序、迁移和 API contract 建立更稳定的质量保障。

## 工作项一：管理端协作与审计增强

### 目标

把管理端从“单人操作工具”升级为“多人协作可追踪的运营后台”，确保关键操作有记录、有影响范围、有回滚依据。

### 实施内容

- 新增操作日志：
  - 记录操作人。
  - 记录操作类型。
  - 记录目标对象类型和 ID。
  - 记录变更前后摘要。
  - 记录操作时间和来源。
- 扩展审核状态：
  - 待审核。
  - 需复核。
  - 已修正。
  - 已发布。
  - 已驳回。
- 批量操作预览：
  - 发布前展示影响的数据条数。
  - 驳回前展示影响的指标、城市和周期。
  - 配置修改前展示关键字段变化。
- 配置变更历史：
  - 指标配置历史。
  - 首页推荐配置历史。
  - 质量规则配置历史。
  - 数据源默认策略历史。
- 回滚能力：
  - 支持恢复某个配置历史版本。
  - 支持查看回滚前后差异。

### 建议交付物

- `operation_logs` 或等价审计模型。
- 操作日志 repository/service。
- 管理端操作日志页面。
- 批量发布/驳回预览 API。
- 配置历史与回滚 API。
- 审计相关测试。

### 验收标准

- 修改指标、配置、数据源、质量规则时能生成操作日志。
- 发布、驳回、修正数据时能看到操作人和操作原因。
- 批量发布或驳回前能看到影响范围。
- 配置历史可以查看，并能恢复历史版本。
- 测试覆盖日志写入、影响预览和配置回滚路径。

## 工作项二：小程序数据解释和用户体验增强

### 目标

把小程序从“展示数据”升级为“解释数据”，让用户理解指标含义、趋势变化和城市对比结果。

### 实施内容

- 指标详情页：
  - 指标口径说明。
  - 单位。
  - 数据来源。
  - 更新时间。
  - 适用范围。
  - 默认维度解释。
- 城市对比体验优化：
  - 固定常用对比城市。
  - 保存城市对比组合。
  - 快速切换对比指标。
  - 对比组合在首页或趋势页优先展示。
- 趋势图体验升级：
  - 标记最高点和最低点。
  - 标记最新一期涨跌。
  - 支持近 3 期、近 6 期、近 12 期快捷筛选。
  - 展示当前值、均值、最大值、最小值。
- 首页信息架构优化：
  - 推荐指标区。
  - 收藏组合区。
  - 热门排行区。
  - 最近更新区。
- 空态和错误态增强：
  - 指标无数据时说明原因。
  - 对比城市不足时提示补充选择。
  - 网络失败时保留缓存数据并提示刷新。

### 建议交付物

- 小程序指标详情页。
- 小程序对比组合保存能力。
- 趋势图统计摘要。
- 首页分区重构。
- 小程序空态、错误态和缓存降级优化。

### 验收标准

- 用户可以从指标卡片进入指标详情。
- 趋势页能保存并恢复城市对比组合。
- 趋势图能展示高低点、最新变化和时间窗口筛选。
- 首页能清晰展示推荐、收藏、排行和更新信息。
- 小程序 smoke test 覆盖指标详情、对比组合和时间窗口切换。

## 工作项三：数据查询与分析能力增强

### 目标

增强后端分析能力，让小程序和管理端不需要自行拼装复杂统计结果。

### 实施内容

- 派生指标：
  - 同比变化。
  - 环比变化。
  - 差值。
  - 排名变化。
  - 距均值偏离。
- 时间窗口查询：
  - 近 3 期。
  - 近 6 期。
  - 近 12 期。
  - 自定义起止周期。
- 区域聚合：
  - 全国。
  - 省份。
  - 城市。
  - 同层级区域对比。
  - 父子区域关联查询。
- 展示聚合接口：
  - 指标详情聚合接口。
  - 趋势摘要接口。
  - 对比分析接口。
  - 首页聚合接口增强。
- 查询性能优化：
  - 常用查询索引。
  - 分页和 limit 控制。
  - 小程序响应字段瘦身。

### 建议交付物

- 分析查询 service。
- 趋势摘要 API。
- 指标详情 API。
- 对比分析 API。
- 相关索引 migration。
- 查询测试。

### 验收标准

- API 可以返回时间窗口内的趋势点和摘要统计。
- API 可以返回同比、环比、差值和排名变化。
- 区域查询可以表达全国、省、市层级关系。
- 小程序不需要多次请求才能完成趋势摘要和对比分析展示。
- 测试覆盖派生指标、时间窗口、区域层级和分页限制。

## 工作项四：配置中心产品化

### 目标

把当前 JSON 编辑式配置升级为表单化、可校验、可预览的配置中心，降低运营误操作风险。

### 实施内容

- 指标配置表单：
  - 展示名称。
  - 分类。
  - 单位。
  - 精度。
  - 排序。
  - 小程序展示开关。
  - 默认维度。
- 首页配置表单：
  - 推荐指标。
  - 推荐城市。
  - 热门排行指标。
  - 默认趋势指标。
  - 分区顺序。
- 质量规则配置表单：
  - 数据源类型。
  - 数值范围阈值。
  - warning/error 分级。
  - 必填维度。
  - 最小数据量。
- 数据源默认策略表单：
  - 默认调度间隔。
  - 默认重试次数。
  - 默认超时时间。
  - 默认质量规则集。
- 配置校验与预览：
  - 字段级校验。
  - 提交前 diff 预览。
  - 保存后刷新生效状态。

### 建议交付物

- 管理端配置中心页面重构。
- 配置 schema 定义。
- 配置校验 API。
- 配置 diff 预览 API。
- 配置表单测试。

### 验收标准

- 运营人员不需要直接编辑 JSON 即可调整核心配置。
- 配置提交前能看到差异和校验结果。
- 错误配置无法保存，且错误信息明确。
- 指标、首页、质量规则、数据源默认策略都有对应表单。
- 测试覆盖合法配置、非法配置、diff 预览和保存路径。

## 工作项五：测试与质量保障增强

### 目标

将当前后端测试和小程序 smoke test 扩展为更完整的质量保障体系，减少后续功能迭代破坏接口和页面行为。

### 实施内容

- 管理端前端测试：
  - 登录态。
  - 数据源详情。
  - 任务详情。
  - 指标配置。
  - 配置保存。
  - 批量审核操作。
- 小程序状态测试：
  - 空态。
  - 错误态。
  - 缓存过期。
  - 收藏恢复。
  - 对比组合恢复。
- migration 测试：
  - 新库 upgrade。
  - 旧库 upgrade。
  - downgrade 或至少 downgrade 语法检查。
- API contract snapshot：
  - 小程序核心 API 响应结构快照。
  - 管理端关键 API 响应结构快照。
  - 避免字段误删或结构漂移。
- 验证脚本整合：
  - 后端测试。
  - ruff。
  - 管理端 build。
  - 小程序 smoke。
  - migration 检查。

### 建议交付物

- 管理端测试框架和首批用例。
- 小程序状态测试扩展。
- migration 验证脚本。
- API contract snapshot。
- 一键质量检查脚本。

### 验收标准

- 管理端关键交互具备自动化测试。
- 小程序空态、错误态、缓存和收藏恢复具备测试。
- migration 能在自动化检查中验证。
- API 响应结构变化会被 snapshot 捕获。
- 一键质量检查可以覆盖后端、管理端、小程序和迁移。

## 推荐里程碑

### 里程碑一：审计与协作基础

优先补齐操作日志、批量操作预览和配置历史，降低多人协作风险。

交付物：

- 操作日志模型。
- 操作日志页面。
- 批量操作预览。
- 配置历史。

### 里程碑二：配置中心表单化

将当前 JSON 配置编辑升级为表单化配置，提升运营可用性。

交付物：

- 指标配置表单。
- 首页配置表单。
- 质量规则配置表单。
- 数据源默认策略表单。

### 里程碑三：小程序解释力增强

让用户能理解指标和趋势，而不只是看到数值。

交付物：

- 指标详情页。
- 趋势摘要。
- 高低点和涨跌标记。
- 对比组合保存。

### 里程碑四：分析查询增强

补齐派生指标、时间窗口和区域聚合能力。

交付物：

- 趋势摘要 API。
- 派生指标 API。
- 区域层级查询。
- 对比分析接口。

### 里程碑五：质量保障体系

最后补齐前端测试、状态测试、migration 检查和 API contract snapshot。

交付物：

- 管理端测试。
- 小程序状态测试。
- migration 检查。
- API contract snapshot。
- 一键质量检查脚本。

## 推荐执行顺序

1. 管理端协作与审计增强。
2. 配置中心产品化。
3. 小程序数据解释和用户体验增强。
4. 数据查询与分析能力增强。
5. 测试与质量保障增强。

优先级理由：先补审计和配置中心，降低后续运营误操作风险；再增强小程序解释力和后端分析能力；最后将新增能力纳入更完整的质量保障体系，避免持续迭代中出现接口漂移和前端行为回退。
