# 数据质量规则

质量检查已改为按数据源类型选择规则。

## 房价数据 `housing_price`

- 期望区域数：70。
- 期望指标数：3。
- 最小值数量：210。
- 必填维度：`house_type`、`area_type`。
- 合法数值范围：0 到 200。
- 常规预警范围：80 到 130。

## CPI 数据 `cpi`

- 期望区域数：1。
- 期望指标数：3。
- 最小值数量：2。
- 必填维度：`source_type`、`frequency`。
- 合法数值范围：-20 到 20。
- 常规预警范围：-10 到 10。

## 发布影响

- error：质量报告为 failed，导入数据进入 `quality_failed`，阻断直接发布。
- warning：质量报告仍可 passed，但报告会保留预警信息，供人工复核。
- `details`：质量报告会记录规则名、严重等级、指标、区域、周期、数值和维度，方便管理端定位具体问题。

后续新增数据源时，应优先在 `QUALITY_RULES` 中补齐规则，再接入 importer。
